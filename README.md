# Concurrency_and_Parallelism
# Python Concurrency & Parallelism

โปรเจกต์นี้เป็นการทดลองเขียนโปรแกรมเพื่อทำความเข้าใจการทำงานแบบพร้อมกัน (Concurrency) และการทำงานแบบคู่ขนาน (Parallelism) ในภาษา Python โดยแบ่งออกเป็น 3 เทคนิค ดังนี้:

## 1. Threading (`thread_task.py`)
- **จำลองสถานการณ์:** การดาวน์โหลดไฟล์ขนาดใหญ่พร้อมกัน
- **ความเหมาะสม:** เหมาะกับงานประเภท **I/O Bound** เช่น รอการโหลดไฟล์จากอินเทอร์เน็ต หรืออ่านเขียนไฟล์ในเครื่อง การใช้ Thread ช่วยให้ไม่ต้องรอโหลดเสร็จทีละไฟล์

## 2. Asyncio (`asyncio_task.py`)
- **จำลองสถานการณ์:** การดึงข้อมูล API ของผู้ใช้หลายคนพร้อมกัน
- **ความเหมาะสม:** เหมาะกับงาน **I/O Bound ที่มีความถี่สูง (High Concurrency)** การใช้ Asyncio (Coroutines) จะสลับการทำงานได้เร็วกว่าและใช้หน่วยความจำน้อยกว่าการสร้าง Thread จำนวนมากๆ

## 3. Process Pool (`process_pool_task.py`)
- **จำลองสถานการณ์:** การคำนวณหาจำนวนเฉพาะ (Prime Numbers) จากตัวเลขหลักล้าน
- **ความเหมาะสม:** เหมาะกับงานประเภท **CPU Bound** (ประมวลผลหนัก) เนื่องจาก Python มีข้อจำกัดเรื่อง GIL (Global Interpreter Lock) ใน Thread การใช้ Process Pool จะเป็นการแตก Process ใหม่ไปยัง CPU คอร์อื่นๆ ทำให้ทำงานขนานกันได้อย่างแท้จริง
